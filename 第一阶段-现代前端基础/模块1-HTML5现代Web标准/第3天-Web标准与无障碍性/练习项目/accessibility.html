什么是Web无障碍性？

Web无障碍性是指设计和开发网站、工具和技术，使残障人士能够使用它们。更具体地说，人们可以感知、理解、导航和与Web交互，并且可以为Web做出贡献。

无障碍性涵盖了影响Web使用的各种残障，包括：

  • 视觉障碍 - 失明、低视力、色盲
  • 听觉障碍 - 失聪、听力困难
  • 运动障碍 - 无法使用鼠标、反应迟缓、精细运动控制有限
  • 认知障碍 - 学习困难、注意力分散、记忆困难

WCAG 2.1 四大原则

1. 可感知性 (Perceivable)

信息和用户界面组件必须以用户能够感知的方式呈现给用户。

示例：图片替代文本

2. 可操作性 (Operable)

用户界面组件和导航必须是可操作的。

示例：键盘导航

尝试使用Tab键在此页面上导航。所有交互元素都应该可以通过键盘访问。

3. 可理解性 (Understandable)

信息和用户界面的操作必须是可理解的。

示例：清晰的表单标签和错误提示

4. 健壮性 (Robust)

内容必须足够健壮，以便各种用户代理（包括辅助技术）都能可靠地解释它。
