📝 版本控制工作流

Git 工作流

# 初始化仓库 git init # 添加文件 git add . # 提交更改 git commit -m "feat: 添加现代工具链示例" # 创建分支 git checkout -b feature/new-component # 推送到远程 git push origin feature/new-component

提交规范

Conventional Commits

  • feat: 新功能
  • fix: 修复bug
  • docs: 文档更新
  • style: 代码格式
  • refactor: 重构
  • test: 测试
  • chore: 构建过程或辅助工具的变动
Git 状态: 准备就绪
