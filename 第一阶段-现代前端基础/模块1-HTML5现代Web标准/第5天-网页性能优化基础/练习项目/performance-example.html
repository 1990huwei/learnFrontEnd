📦 资源加载优化

通过各种预加载技术和资源优化策略来提升页面性能。

预加载 (Preload)

预加载当前页面需要的关键资源，如关键CSS、字体、重要图片等。

<link rel="preload" href="critical.css" as="style">

预连接 (Preconnect)

提前建立与重要第三方域名的连接，减少DNS查询和连接建立时间。

<link rel="preconnect" href="https://fonts.googleapis.com">

DNS预解析 (DNS-prefetch)

提前解析第三方域名的DNS，减少后续请求的延迟。

<link rel="dns-prefetch" href="//cdn.example.com">

关键CSS内联

将首屏渲染所需的CSS直接内联到HTML中，避免额外的网络请求。

<style>/* 关键CSS */</style>

异步CSS加载

非关键CSS异步加载，避免阻塞页面渲染。

<link rel="preload" href="style.css" as="style" onload="this.rel='stylesheet'">

JavaScript优化

使用async和defer属性优化JavaScript加载，避免阻塞HTML解析。

<script src="script.js" async></script>

📊 Core Web Vitals 优化

Core Web Vitals是Google用来衡量用户体验的核心指标，直接影响SEO排名。

LCP - 最大内容绘制

目标：< 2.5秒

优化方法：

  • 优化服务器响应时间
  • 预加载关键资源
  • 优化图片和视频
  • 移除阻塞渲染的资源

FID - 首次输入延迟

目标：< 100毫秒

优化方法：

  • 减少JavaScript执行时间
  • 拆分长任务
  • 使用Web Workers
  • 优化第三方代码

CLS - 累积布局偏移

目标：< 0.1

优化方法：

  • 为图片指定尺寸
  • 避免在现有内容上方插入内容
  • 使用transform动画
  • 预加载字体
